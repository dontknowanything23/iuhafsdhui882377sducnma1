Min_Run_Min_Stop

Path: CARNOT/Control


Purpose:
Assures a minimum operation and a minimum stop time.

Description:
The block keeps the control signal:

  • At or above the minimum output value for the minimum operation time.
  • To zero for at least the minimum stop time.

The block is a combination of Min_Run_Time and Min_Stop_Time.
Special case : Starting the simulation with an input of 1 is interpreted as a control signal which is already 1 since a long time (= longer than the minimum operation time).

Input:

In1 : control signal (no unit)

Output:

Out1 : control signal (no unit)

Parameters and Dialog Box:

Characteristics:

Direct Feedthrough : Yes
Sample Time : Inherited from driving block
Vectorized : No
